# Stage 1: `market_daily`

来源飞书文档：
<https://jcng9kabbb3s.feishu.cn/docx/Q6ted4WxJoRfBQx1Tdlcdyh0nId>

## 目标

`market_daily` 用于记录 `SHFE CU` 各真实合约的日度行情数据，作为后续研究、日报和上下文生成的底层市场数据表。

## 字段定义

| 字段名 | 类型建议 | 必需 | 说明 | 示例 |
| --- | --- | --- | --- | --- |
| `date` | `date` | 是 | 交易日 | `2026-03-28` |
| `symbol` | `string` | 是 | 品种代码，第一阶段固定为 `CU` | `CU` |
| `contract` | `string` | 是 | 真实合约代码 | `CU2505` |
| `open` | `numeric` | 是 | 开盘价 | `81120` |
| `high` | `numeric` | 是 | 最高价 | `81380` |
| `low` | `numeric` | 是 | 最低价 | `80940` |
| `close` | `numeric` | 是 | 收盘价 | `81230` |
| `settlement` | `numeric` | 是 | 结算价 | `81190` |
| `volume` | `numeric` | 是 | 成交量，单位保持交易所原始口径 | `156789` |
| `open_interest` | `numeric` | 是 | 持仓量，单位保持交易所原始口径 | `198765` |
| `source` | `string` | 是 | 数据来源 | `SHFE` |
| `fetched_at` | `datetime` | 是 | 系统抓取并入库时间 | `2026-03-28 15:35:10` |
| `note` | `string` | 否 | 异常说明或补充备注 | `结算价缺失，已重试一次` |

## 补充说明

- `settlement` 是交易所用于当日结算和盯市的官方价格口径，不一定等于最后一笔成交形成的 `close`。
- 保留 `settlement` 是为了区分“市场最后收在哪”和“交易所按什么价格结算”。
- 上期所 `CU` 合约通常维持 12 个连续月份在市，但这是目标数量，不是硬约束。

## 字段保留原因

| 字段名 | 需求原因 |
| --- | --- |
| `date` | 作为日度时间轴，支撑查询、回看和多表按日对齐 |
| `symbol` | 保留统一品种维度，为后续扩品种和统一查询留标准位 |
| `contract` | 保留真实合约身份，避免后续研究视角失去可追溯性 |
| `open` | 支撑开盘位置判断 |
| `high` | 支撑盘中上沿和波动判断 |
| `low` | 支撑盘中下沿和波动判断 |
| `close` | 作为最核心的收盘价格锚 |
| `settlement` | 保留期货正式结算口径，方便后续规范比较 |
| `volume` | 判断价格变化是否有交易参与 |
| `open_interest` | 判断资金是否持续进场或离场 |
| `source` | 支撑数据审计、排错和来源治理 |
| `fetched_at` | 支撑抓取任务审计和数据时效检查 |
| `note` | 记录异常、缺失和补充说明，避免把边角情况硬塞进主字段 |

## 记录粒度

一条记录表示某个交易日、某个真实合约的一条日度行情数据。

示例：

- `2026-03-28 + CU2504`
- `2026-03-28 + CU2505`
- `2026-03-28 + CU2509`

## 校验规则

| 规则 | 要求 |
| --- | --- |
| 主键约束 | 同一 `date + symbol + contract` 只能有一条有效记录 |
| 日期口径 | `date` 使用交易日，不使用抓取日 |
| 品种口径 | `symbol` 第一阶段固定为 `CU` |
| 合约口径 | `contract` 必须使用交易所真实合约代码 |
| 数值口径 | `open/high/low/close/settlement/volume/open_interest` 必须为数值类型 |
| 价格约束 | `high >= low`，`open/close/settlement` 应落在合理价格区间内 |
| 非负约束 | `volume >= 0`，`open_interest >= 0` |
| 来源口径 | `source` 使用稳定字符串，不混用别名 |
| 时间口径 | `fetched_at` 记录系统抓取入库时间 |

## 异常处理

| 场景 | 处理方式 |
| --- | --- |
| 非交易日 | 不生成记录 |
| 单字段缺失 | 记录异常并写入 `note`，同时保留抓取日志 |
| 全行缺失 | 不伪造数据，记录缺失并触发补抓 |
| 重复记录 | 按 `date + symbol + contract` 去重，只保留一条有效记录 |
| 数值异常 | 不静默吞掉，记录异常并进入校验结果 |
| 来源切换 | 更新 `source`，不得沿用旧值伪装同口径 |

## 交付标准

| 项目 | 完成标准 |
| --- | --- |
| 抓取 | 能按交易日稳定抓取 `SHFE CU` 合约日度行情 |
| 落库 | 能稳定写入 `market_daily` |
| 查询 | 能按 `date / symbol / contract` 查询记录 |
| 样本 | 能输出至少 30 个交易日样本用于验数 |
| 校验 | 能输出缺失、重复、异常值的基础校验结果 |
| 可追溯 | 每条记录都能追溯到 `source` 和 `fetched_at` |

## SQLite 落地建议

- 表名使用 `market_daily`
- 主键或唯一约束使用 `date, symbol, contract`
- `date` 使用 ISO 日期字符串或 SQLite `DATE`
- `fetched_at` 使用 ISO 时间字符串或 SQLite `DATETIME`
- 金额和数量字段在 Python 入库前先做显式数值转换，不依赖 SQLite 隐式类型
